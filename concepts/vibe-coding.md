# Vibe-Coding

> "전통적인 소프트웨어 엔지니어링 배경 없이도, 심리학과 도메인 지식을 기반으로 AI와 협업하여 실제 작동하는 시스템을 구축하는 아키텍처적 태도."

---

## 정의 (Compiled Truth)

**Vibe-Coding**은 코드를 직접 한 줄 한 줄 작성하는 대신, 해결하려는 문제의 **'맥락(Context)'**과 **'의도(Intent)'**를 AI에게 정교하게 전달하여 소프트웨어를 생성하는 방식입니다. 이는 단순한 '코드 복사-붙여넣기'가 아니라, 시스템의 구조와 논리적 흐름을 설계하는 **'심리적 설계'**에 가깝습니다.

### 핵심 원칙
1.  **Context-First**: 언어적 모호성을 줄이고 AI가 문제를 정확히 이해할 수 있도록 풍부한 배경 지식을 제공합니다.
2.  **Intent-Driven**: 구현 방법(How)보다 달성하려는 목적(What/Why)에 집중합니다.
3.  **Iterative Refinement**: 생성된 결과를 즉시 테스트하고, 발견된 결함을 기반으로 AI와 대화하며 시스템을 정교화합니다. (Red-Green-Refactor with AI)
4.  **Phenomenological Thinking**: 이론적 추상화보다 실제 사용자 경험과 현장의 언어를 우선합니다.

## CSP의 Vibe-Coding 철학

1.  **심리학이 코드가 된다**: 리더십 이론(LMX), 몰입 이론(SDT) 등 인문학적 프레임워크를 조건문과 데이터 구조로 변환합니다.
2.  **철학이 알고리즘이 된다**: "나답게 살아도 두려움 없는 환경"과 같은 실존적 가치를 시스템의 최종 지향점으로 설정합니다.
3.  **제로-터치 자동화**: 인간의 개입을 최소화하면서도 도메인 전문가의 통찰이 반영된 자동화 시스템을 지향합니다.

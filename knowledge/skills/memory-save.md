# 스킬: 메모리 저장 (Memory Save)

> 대화 종료 후 핵심 내용을 Brain에 기록하는 스킬.

---

## 언제 쓰는가

- 의미 있는 대화/작업 세션이 끝날 때
- 중요한 결정이 내려졌을 때
- 새로운 개념을 배웠을 때
- 프로젝트 상태가 변화했을 때

---

## 실행 단계

1. **Timeline append**
   - 관련 `projects/[이름]/README.md` 열기
   - 오늘 날짜(`### YYYY-MM-DD`) 섹션 추가
   - 핵심 내용 bullet로 기록 (3~7개)

2. **Compiled Truth 업데이트** (필요 시)
   - 상태, 다음 액션, 핵심 결정사항 변경 시 덮어씀

3. **파생 파일 생성** (해당 시)
   - 새 결정 → `decisions/[YYYY-MM-DD]-[결정명].md`
   - 새 개념 → `concepts/[개념명].md`
   - 새 인물 → `people/[이름].md`

4. **미분류 처리**
   - 어디 넣을지 모르겠으면 → `raw/inbox.md`에 태그와 함께 추가

---

## Timeline 작성 원칙

```markdown
### 2026-04-14
- [결정] 무엇을 결정했는가
- [학습] 무엇을 새로 알게 됐는가
- [진행] 어떤 작업을 수행했는가
- [문제] 어떤 이슈가 발생했는가
- 다음 액션: [구체적인 다음 단계]
```

---

## 주의사항

- Timeline은 append만 — 절대 이전 항목 수정/삭제 금지
- 너무 상세하게 쓰지 않아도 됨 — 핵심만
- 감정/주관적 판단도 기록 가능 (나중에 패턴 파악에 도움)

---

## 관련 프롬프트

```
오늘 대화 내용을 csp-brain에 저장해줘.
오늘 날짜: [날짜]
관련 프로젝트: [프로젝트명]
```
